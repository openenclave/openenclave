401 /* OP-TEE only allows one thread per TA to be in an ecall. Even if it has
402  * an ocall in progress, that ecall must complete before another ecall
403  * can enter the TA. SGX, on the other hand, would allow a second ecall
404  * to enter. So to allow them to function identically, apps should wrap
405  * ecalls in the following mutex Acquire/Release calls. In the future,
406  * if we have our own code generator instead of sgx_edger8r, these could
407  * be automatic instead of requiring manual coding effort to call.
408  */
